# AbeloHost Shop

### Getting Started

First, run the development server:

```bash
npm install
cp .env.example .env
npm run dev
```

### Technical stack

- TypeScript
- NextJS (App Router)
- Zustand
- Axios
- SCSS-модули
- Prettier
- ESLint
- Stylelint

### Folder structure

- FSD

### API for requesting data

- API DummyJSON (https://dummyjson.com/)

### Building and launching the application

- Docker

```bash
# build
docker build -t abelohost-shop .

# start
docker run -p 3001:3000 --rm abelohost-shop
```

## Ответы на вопросы

#### 1. Какие сложности, если они были, возникли при выполнении тестового задания?

Ответ:\
a) Тежело было соблюдать pixel-perfect из-за плохого качества исходного изображения, и не смог опредлить шрифт, похож на Montserrat, но есть отличия.\
b) Были трудности с настройкой API для авторизации. Изначально сделал на стороне клиента, но понял, что разумнее на стороне сервера

#### 2. Как вы оцениваете полноту и качество своей реализации?

Ответ: Считаю, что задание выполнено на 8 из 10. Возможно неидеально настроил SSR и правила в линтерах.

#### 3. Если бы это был production-проект и у вас было больше времени, какие изменения или доработки вы бы внесли?

Ответ:\
a) Глубже бы занялся оптимизаций (кеширование запросов, настройки ssr)\
b) Больше времени потратил бы на тестирование в разных браузерах и мобильных клиентах.\
c) Написал бы тесты.\
d) Если рассматривать проект как полноценный интернет-магазин, то следует реализовать все базовые функции такого проекта - корзина, оплата, аналитика, мониторинг ошибок и прочее. UX доработки - темная тема, локализация. SEO доработки - метатеги, ключевые слова.\
e) Не успел написать логику по обновлению рефреш токена\
f) Не успел сделать пагинацию товаров

#### 4. Уточните пожалуйста, использовали ли вы ИИ при выполнении и если использовали, то для чего конкретно. Это важно для корректной оценки.

Ответ: В рамках задания не использовал AI агентов для написания кода. Использовал только AI-автокомплит, чтобы оптимизировать скорость написания базовых вещей типа map() списков, JSX разметка и css правила, и прочий мелкий код. Все что забыл или вызвало трудности гуглил в поиске (например eslint правила).
